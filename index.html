À propos

Méthode, rigueur, et un goût pour le design propre.

Approche

J’avance comme une enquête : je capture le comportement, je réduis le bruit, j’isole les points clés, puis je documente clairement.

  • Observe → logs, traces, diff
  • Extract → classes, strings, calls, assets
  • Understand → modèles, contrôles, flux
  • Report → writeup, preuves, mitigations
